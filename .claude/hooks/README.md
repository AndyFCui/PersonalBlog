# Hooks

目标：把规则从“建议”变成“执行前检查”。

## 设计原则

- pre-task：做任务前检查上下文
- post-edit：改完代码后检查文档同步
- pre-commit：提交前阻断明显违规项

## 铁律

1. 发现业务文件缺少 L3 头部，阻塞
2. 发现架构变更但未更新 CLAUDE.md，阻塞
3. 发现新增逻辑却无测试建议，警告
4. 发现高风险操作，要求人工确认

## 注意

具体 hook 接入方式取决于你的运行环境。
这里提供的是一套通用脚本模板，你可以映射到：
- git hooks
- CI checks
- Claude Code 自定义命令链
- 本地脚本入口
